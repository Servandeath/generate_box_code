# generate_box_code

Генератор уникальных кодов коробов (упаковки) для Wildberries -
десктоп-инструмент с локальной БД для проверки уникальности уже
напечатанных кодов.

## Проблема и задача

С появлением привязки штрихкода короба к конкретному городу/складу
в системе Wildberries возникла операционная проблема: короба,
промаркированные заранее или на другом складе, требуют перемаркировки
при несовпадении с текущей точкой отгрузки. Это увеличивает количество
ручных операций и время на передачу данных о ШК складу.

В качестве решения тестируется подход с самостоятельной генерацией
кодов коробов, не привязанных к конкретному складу на момент создания.
Код формируется прямо во время сборки заказа и заносится в локальную
таблицу учёта, без ожидания подтверждения от склада или системы WB.

**Ожидаемый эффект:** снижение операционных затрат времени на 15-30%
в зависимости от используемой системы мониторинга и планирования
поставок и их частотности - за счёт исключения шага перемаркировки
и сокращения ручных передач данных между этапом сборки и складом.

## Требования Wildberries к коду короба

- длина 6-30 символов
- не начинается с WB
- без пробелов
- только латинские буквы, цифры, дефис, подчёркивание
- уникальность регистронезависимая (AbCdEfG = abcdefg)
- рекомендуется делать код длинным и сложным - риск случайного
  совпадения с чужим кодом действует на всей платформе, не только
  внутри одного продавца

## Формат кода

CABINET_dd_MM_YYYY_SEASON_ITEM_RANDOMSEQ

Пример: MAN_16_07_2026_DE_BT_R4N001

- CABINET - код кабинета из справочника (справочник редактируемый, не хардкод)
- dd_MM_YYYY - дата генерации
- SEASON - код сезона, редактируемый справочник
- ITEM - код типа предмета, редактируемый справочник
- RANDOMSEQ - без разделителя внутри: случайные буквенно-цифровые символы
  + порядковый номер (число цифр зависит от max_seq, см. ниже)

### Стандартная длина кодов в справочниках

- CABINET - первые 3 символа (МАНИ -> MAN, МИРОС -> MIR, МЕЛУА -> MEL)
- SEASON - 2 символа
- ITEM - 2 символа, предпочтительно согласные для читаемости (BT, LF, TF, BL)

Это не жёсткое ограничение на уровне generate_box_code() (функция
принимает любые валидные строки), а рекомендуемый стандарт для слоя
транслитерации при сохранении записи в справочник - он гарантирует
достаточный запас случайной части даже при max_seq до 4 цифр.

### Динамическая длина случайной части

Случайная часть - не фиксированная константа. Генератор считает,
сколько символов уже занято (кабинет + дата + сезон + предмет +
разделители + цифры порядкового номера), и добирает случайными
символами остаток до лимита 30. Есть нижний порог (минимум 3 символа
случайной части) - если бюджета не остаётся, генератор возвращает
ошибку, а не код без энтропии.

### Порядковый номер

Общий счётчик на кабинет за сутки (сбрасывается на новый день),
не зависит от категории или сезона. Верхняя граница (max_seq) -
параметр, не константа: ожидаемый объём коробов в сутки на кабинет
может быть 300, может быть 1000+. Число цифр номера считается как
len(str(max_seq)) и вычитается из бюджета случайной части.

## Архитектура (по слоям)

1. БД (SQLite) - справочники cabinets, seasons, item_types
   (с флагом is_active для мягкого отключения без потери истории),
   основная таблица box_codes - история сгенерированных кодов
2. Справочники (CRUD) - редактирование через GUI, поле ввода
   принимает кириллицу/латиницу/цифры, транслитерация в латинский код
   с возможностью ручной правки перед сохранением
3. Генератор - выбор кабинета/сезона/предмета из кэша, дата,
   количество -> черновик в памяти -> предпросмотр -> подтверждение ->
   запись в БД
4. Этикетка - штрихкод Code128 (не EAN-13 - код буквенно-цифровой,
   переменной длины), настраиваемые позиции текста/ШК на этикетке,
   крупный шрифт для читаемости номера издалека, печать по одному
   файлу или пачкой (1 страница = 1 ШК)
5. Экспорт - Excel, список сгенерированных кодов (1 колонка)

## Стек

- Python 3.11+
- GUI: PySide6 (табличный интерфейс, QTableView для истории кодов)
- БД: SQLite
- Штрихкод/PDF: reportlab (Code128)
- Excel: openpyxl

## Статус

MVP в разработке. Реализовано: чистая функция generate_box_code
с юнит-тестами на граничные случаи, стандарт длины кодов справочников.
Дальше: транслитерация кириллицы, схема БД, GUI-слои.

## Установка

pip install -r requirements.txt

## Требования к этикетке (зафиксировано)

- Порядковый номер (последние 3+ цифры кода, RANDOMSEQ) должен печататься
  увеличенным шрифтом относительно остального текста на этикетке - для
  считывания издалека без сканера. Это отдельная настройка размера шрифта,
  не общий font_size для всех текстовых полей этикетки.
